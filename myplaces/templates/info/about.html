{% trans 'About MyPlaces'%}

{%blocktrans%} MyPlaces is a free project that enables to present collections of places on a map. Many similar services exists, but we focus on following values:
  • Simplicity
    Our user interface is minimalistic and simple, focused around core functionality of the service.
  • Mobility
    MyPlaces site is using responsible page design so it can be accessed from modern mobile phones, where it provides same interface, but optimized for smaller screens.
  • Openness
    MyPlaces is fully based on Open Source technologies. Sources for itself are available on github.
    Also used maps are from fantastic OpenStreetMap project.
  • Passion
    I like maps. I like to browse them, see their various features etc. That's why I've learned a bit about GIS and current map technologies on Internet. This site is partially my playground for these technologies and partially place to see collections of places in an easy way.
  • Extra
    We would like to provide something extra, which is not very common in similar services either in areas of geographic data visualization and functionality. As per now we can offerer an interesting visualization - Voronoi diagram of places in a collection, or flexible import functionality, where a collection can be imported from several formats (CSV, GPX, GeoJSON). I hope we will be able to provide more in future.
{%endblocktrans%}

{% trans 'Technology'%}

{%blocktrans%} The service is build in Python programming language, using Django (and GeoDjango supporting various geographic operations). Python code is running in gevent webserver (with support for websockets and socket.io protocols) as a backend service behind nginx webserver, which provides static data, load balancing to backends and caching.
Backend uses Postgres database to store and manipulate data and PostGIS extension to this database.
Application is designed (mostly) as a single page application leveraging RESTful services on backend servers (with help of django-rest-framework) and backbone.js and jQuery Javasript libraries on client side to provide dynamic interface. {%endblocktrans%}
